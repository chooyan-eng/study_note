# study_note

## ドキュメント

このプロジェクトで作業する際は必ず以下を参照すること。

@docs/SPEC.md
@docs/TECH_NOTES.md
@docs/TASKS.md

## ドキュメント自動更新ルール

対話の中で以下に該当する変更が生じた場合、**作業完了後に必ず対応するドキュメントを更新すること**。ユーザーへの確認は不要。

### 更新対象と条件

| ドキュメント | 更新するケース |
|---|---|
| `docs/SPEC.md` | 機能仕様・要件・UI設計・データモデル・フェーズ計画に変更や追加があった場合 |
| `docs/TECH_NOTES.md` | ライブラリの使い方・実装方針・アーキテクチャの詳細に変更や追加があった場合 |

### 更新方針

- 変更点のみを差分更新する（ドキュメント全体の書き直しは不要）
- 既存の記述と矛盾する内容は古い記述を削除または上書きする
- 新しいセクションが必要な場合は既存の構造・スタイルに合わせて追記する
- 更新後、変更した箇所をユーザーに簡潔に報告する

## 動作確認ルール

実装タスク完了後は必ず以下のコマンドで簡易動作確認を行うこと。

```
fvm flutter run -d macos
```

- `fvm` 経由で実行すること（`flutter` コマンドを直接使用しない）
- 起動確認後はアプリを停止してよい
- エラーが出た場合はその場で修正してから報告する
